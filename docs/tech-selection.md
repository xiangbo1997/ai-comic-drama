## 2. 各环节技术选型

### 2.1 文本/剧本处理 (LLM)

| 方案 | 选型 | 理由 |
|------|------|------|
| **首选** | DeepSeek V3 | 中文理解强、便宜、速度快 |
| **备选** | GPT-4o | 复杂任务表现好 |
| **兜底** | Claude 3.5 | 长文本处理 |

**成本**: ~¥0.01/1000 tokens (DeepSeek)

### 2.2 角色一致性 (核心难点)

| 优先级 | 方案 | 用途 | 成本 |
|--------|------|------|------|
| **首选** | Flux Kontext Pro | 主力生成 | ~$0.03/张 |
| **备选** | PuLID | 高保真场景 | ~$0.02/张 |
| **预览** | IP-Adapter FaceID | 低成本预览 | ~$0.01/张 |

**策略**: 分层生成 - 预览用低成本，正式用高质量

### 2.3 视频生成

| 优先级 | 方案 | 用途 | 成本 |
|--------|------|------|------|
| **首选** | Runway Gen-3 Turbo | 质量/速度平衡 | ~$0.25/5秒 |
| **备选** | Luma Dream Machine | API开放度高 | ~$0.20/5秒 |
| **兜底** | Replicate SVD | 成本最低 | ~$0.10/5秒 |

**策略**: 主用 Runway，失败降级到 Luma，再降级到 SVD

### 2.4 语音合成 (TTS)

| 场景 | 方案 | 理由 | 成本 |
|------|------|------|------|
| **中文配音** | 火山引擎 | 质量高、便宜 | ~¥0.002/字 |
| **英文/情感** | ElevenLabs | 情感表达好 | ~$0.30/1000字 |
| **备选** | MiniMax | 性价比高 | ~¥0.01/100字 |

## 3. 成本模型

### 3.1 单条漫剧成本 (20个分镜，3分钟)

| 环节 | 单价 | 数量 | 成本 |
|------|------|------|------|
| 剧本拆解 (LLM) | ¥0.01/1K tokens | 5K | ¥0.05 |
| 角色一致性图像 | $0.03/张 | 20张 | $0.60 |
| 视频生成 | $0.25/5秒 | 20段 | $5.00 |
| 语音合成 | ¥0.002/字 | 700字 | ¥1.40 |
| **总计** | | | **~$6.00 (¥45)** |

### 3.2 成本构成分析

```
视频生成 83% ████████████████████
图像生成 10% ██
语音合成  5% █
LLM处理   2%
```

**结论**: 视频生成是最大成本项，需重点优化

### 3.3 成本优化策略

1. **分层生成**: 预览用低质量，确认后再高质量生成
2. **缓存复用**: 相同角色/场景的图像可复用
3. **视频降级**: 非关键场景用静态图+Ken Burns效果代替
4. **批量折扣**: 与服务商谈批量价格

## 4. 技术风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 角色崩坏 | 高 | 高 | 人脸相似度校验 + 自动重试 |
| API不稳定 | 中 | 高 | 多服务商备份 + 降级机制 |
| 成本超预期 | 中 | 中 | 分层生成 + 积分控制 |
| 生成速度慢 | 高 | 中 | 异步队列 + 并行生成 |
| 内容合规 | 中 | 高 | 接入审核API + 人工复核 |
